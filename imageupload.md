# Backend Implementation Prompt: Background Image Upload System with SSE

## Context & Current State

We have a React Native messaging app with a Node.js backend. Currently, media uploads go through a `media.controller` → `media.service` pipeline where:

- Files are received via multer (`req.file.buffer`) and proxied through the server to S3
- Access control checks membership via `Group.findOne()` + `group.isMember(userId)`
- Media IDs are generated server-side, and the S3 key follows the pattern `media/{chatId}/{mediaId}.bin`
- The app uses **SSE** (not WebSockets) for real-time server→client communication
- There is **no end-to-end encryption** — the server can see and process plaintext image files

## What Needs to Change

We need to move from "proxy bytes through the server" to a **presigned URL direct-to-S3 upload** model that supports:

1. **Background uploads** — uploads must survive the React Native app being killed
2. **Multiple image selection** — a user can pick N images for a single message
3. **Idempotency** — no duplicate uploads if the client retries
4. **Failure recovery** — server reconciles orphaned or failed uploads
5. **Non-blocking server-side image processing** — the server generates thumbnails and optimized versions via BullMQ workers, never blocking the API response
6. **SSE-based status notifications** — the server pushes upload/processing status updates to clients via SSE

Because the server can see plaintext images, the BullMQ worker handles real processing: generating thumbnails, creating optimized/compressed versions, stripping EXIF metadata, and storing processed variants back to S3.

## Detailed Implementation Requirements

### 1. Database: Upload Record Schema

Create a new model/collection to track each upload's lifecycle.

**Model: `MediaUpload`**

| Field | Type | Description |
|-------|------|-------------|
| `imageId` | `String, unique, required, indexed` | UUID generated by the client. This is the **idempotency key**. |
| `messageId` | `String, required, indexed` | Links this image to a specific message (client generates this too). |
| `chatId` | `ObjectId, required, indexed` | The group/chat this image belongs to. |
| `userId` | `ObjectId, required` | The uploader. |
| `s3Key` | `String, required` | The raw upload S3 key: `uploads/{chatId}/{imageId}` (with original extension). |
| `thumbnailS3Key` | `String` | Set after processing: `thumbs/{chatId}/{imageId}.webp` |
| `optimizedS3Key` | `String` | Set after processing: `optimized/{chatId}/{imageId}.webp` |
| `status` | `String, enum` | One of: `pending`, `uploaded`, `processing`, `completed`, `failed` |
| `mimeType` | `String` | The actual MIME type of the image (e.g., `image/jpeg`, `image/png`). |
| `sizeBytes` | `Number` | Original file size (sent by client during init). |
| `width` | `Number` | Original image width in pixels (set after processing). |
| `height` | `Number` | Original image height in pixels (set after processing). |
| `thumbnailUrl` | `String` | CDN/API URL for the thumbnail (set after processing). |
| `optimizedUrl` | `String` | CDN/API URL for the optimized image (set after processing). |
| `presignedUrlExpiresAt` | `Date` | When the last-issued presigned URL expires. Used by reconciliation. |
| `createdAt` | `Date` | When the upload was initiated. |
| `updatedAt` | `Date` | Last status change. |

**Indexes:**

- `{ imageId: 1 }` — unique, for idempotent lookups
- `{ messageId: 1, status: 1 }` — to query "are all images for this message complete?"
- `{ status: 1, createdAt: 1 }` — for the reconciliation cron to find stale `pending` records

**Mongoose Schema:**

```javascript

const mediaUploadSchema = new mongoose.Schema({

  thumbnailS3Key:       { type: String, default: null },
  optimizedS3Key:       { type: String, default: null },
  status:               { type: String, enum: ['pending', 'uploaded', 'processing', 'completed', 'failed'], default: 'pending' },

  sizeBytes:            { type: Number },
  width:                { type: Number, default: null },

});

mediaUploadSchema.index({ messageId: 1, status: 1 });
```

### 2. API Endpoints

#### 2.1 `POST /v1/media/upload/init` — Request a Presigned Upload URL

**Purpose:** The client calls this *before* starting each upload. The server creates a DB record and returns a presigned S3 PUT URL. The server does NOT receive any file bytes here — multer is not involved.

**Request body:**
```json
{
  "chatId": "abc123",
  "messageId": "msg-uuid-from-client",
  "imageId": "img-uuid-from-client",
  "mimeType": "image/jpeg",
  "sizeBytes": 4500000
}
```

**Logic (step by step):**

1. Authenticate the request (existing auth middleware).
2. Validate required fields: `chatId`, `messageId`, `imageId`, `mimeType`, `sizeBytes`.
3. **Validate mimeType:** Only allow image types: `image/jpeg`, `image/png`, `image/webp`, `image/heic`, `image/heif`, `image/gif`. Reject anything else with 400.
4. **Membership check:** Find the group by `chatId`, verify `group.isMember(userId)`. Throw `E2E_004` if not found or not a member (keep existing error codes for backward compat).
5. **Size check:** If `sizeBytes > MAX_FILE_SIZE` (100 MB), throw `E2E_005`.
6. **Idempotency check:** Look up `MediaUpload.findOne({ imageId })`.
   - If found with `status: 'completed'` → return `{ alreadyComplete: true, imageId, thumbnailUrl, optimizedUrl }` immediately.
   - If found with `status: 'pending'` or `status: 'failed'` → proceed to step 7 (re-issue a presigned URL for retry).
   - If not found → proceed to step 7 (new upload).
7. **Determine S3 key and extension:**
   ```javascript
   const ext = {
     'image/jpeg': '.jpg',
     'image/png': '.png',
     'image/webp': '.webp',
     'image/heic': '.heic',
     'image/heif': '.heif',
     'image/gif': '.gif',
   }[mimeType] || '.jpg';

   const s3Key = `uploads/${chatId}/${imageId}${ext}`;
   ```
8. **Upsert the DB record:**
   ```javascript
   const expiresAt = new Date(Date.now() + 3600 * 1000); // 1 hour

   await MediaUpload.findOneAndUpdate(
     { imageId },
     {
       $set: {
         messageId, chatId, userId, s3Key, mimeType,
         sizeBytes, status: 'pending', presignedUrlExpiresAt: expiresAt,
         updatedAt: new Date(),
       },
       $setOnInsert: { createdAt: new Date() },
     },
     { upsert: true, new: true }
   );
   ```
9. **Generate presigned PUT URL:**
   ```javascript
   import { getSignedUrl } from '@aws-sdk/s3-request-presigner';

   const command = new PutObjectCommand({
     Bucket: BUCKET_NAME,
     Key: s3Key,
     ContentType: mimeType,
     ContentLength: sizeBytes,
     Metadata: {
       chatId: chatId.toString(),
       uploaderId: userId.toString(),
       imageId,
     },
   });

   const presignedUrl = await getSignedUrl(s3Client, command, { expiresIn: 3600 });
   ```
10. **Return response:**
    ```json
    {
      "success": true,
      "data": {
        "presignedUrl": "https://your-bucket.s3.ap-south-1.amazonaws.com/...",
        "s3Key": "uploads/abc123/img-uuid.jpg",
        "imageId": "img-uuid-from-client",
        "expiresIn": 3600
      }
    }
    ```

#### 2.2 `POST /v1/media/upload/complete` — Confirm Upload Finished

**Purpose:** After the client successfully PUTs to S3, it calls this to tell the server "the file is there, process it."

**Request body:**
```json
{
  "imageId": "img-uuid-from-client"
}
```

**Logic (step by step):**

1. Authenticate.
2. Find the `MediaUpload` record by `imageId`. Return 404 if not found.
3. **Ownership check:** Ensure `upload.userId` matches `req.user._id`. Return 403 if not.
4. **Idempotency:** If `status === 'completed'`, return the existing result immediately:
   ```json
   { "success": true, "data": { "imageId": "img-uuid", "status": "completed", "thumbnailUrl": "...", "optimizedUrl": "..." } }
   ```
   If `status === 'processing'`, return:
   ```json
   { "success": true, "data": { "imageId": "img-uuid", "status": "processing" } }
   ```
5. **Verify S3 object exists** (HeadObject — lightweight, no download):
   ```javascript
   try {
     await s3Client.send(new HeadObjectCommand({
       Bucket: BUCKET_NAME,
       Key: upload.s3Key,
     }));
   } catch (err) {
     if (err.name === 'NotFound' || err.$metadata?.httpStatusCode === 404) {
       return res.status(400).json({ success: false, error: 'File not found in storage' });
     }
     throw err;
   }
   ```
6. **Update status to `uploaded`:**
   ```javascript
   await MediaUpload.updateOne(
     { imageId },
     { $set: { status: 'uploaded', updatedAt: new Date() } }
   );
   ```
7. **Enqueue BullMQ job:**
   ```javascript
   await mediaQueue.add(
     'process-image',
     {
       imageId,
       s3Key: upload.s3Key,
       chatId: upload.chatId.toString(),
       userId: upload.userId.toString(),
       messageId: upload.messageId,
       mimeType: upload.mimeType,
     },
     {
       jobId: imageId,       // BullMQ deduplicates by jobId
       attempts: 3,
       backoff: { type: 'exponential', delay: 2000 },
     }
   );
   ```
8. **Return immediately** (don't wait for the job):
   ```json
   { "success": true, "data": { "imageId": "img-uuid", "status": "processing" } }
   ```

#### 2.5 `GET /v1/media/:mediaId` — Download Media (extend existing)

Keep your existing download endpoint. Extend it to support a `variant` query param:

```
GET /v1/media/img-uuid?variant=thumbnail    → serves the thumbnail
GET /v1/media/img-uuid?variant=optimized    → serves the optimized version (default)
GET /v1/media/img-uuid?variant=original     → serves the raw upload
```

Reconstruct the S3 key based on the variant, verify chat membership, and either redirect to a presigned GET URL or stream the content. Presigned GET redirect is preferred for performance — the client downloads directly from S3/CloudFront instead of proxying through your server.

#### 3.2 Worker Definition — Image Processing with Sharp

The worker downloads the raw image from S3, generates a thumbnail and an optimized version, uploads both back to S3, updates the DB, and notifies via SSE.

```javascript
// workers/media.worker.js
import { Worker } from 'bullmq';
import sharp from 'sharp';
import { S3Client, GetObjectCommand, PutObjectCommand, HeadObjectCommand } from '@aws-sdk/client-s3';
import { MediaUpload } from '../models/MediaUpload.js';
import { Group } from '../models/Group.js';
import { emitSSEEvent } from '../sse/emitter.js';

const REDIS_CONNECTION = {
  host: process.env.REDIS_HOST || '127.0.0.1',
  port: parseInt(process.env.REDIS_PORT || '6379'),
  password: process.env.REDIS_PASSWORD || undefined,
  maxRetriesPerRequest: null,
};

const BUCKET_NAME = process.env.S3_BUCKET_NAME;

const mediaWorker = new Worker(
  'media-processing',
  async (job) => {
    const { imageId, s3Key, chatId, userId, messageId, mimeType } = job.data;

    // ──────────────────────────────────────────────
    // 1. Idempotency: skip if already completed
    // ──────────────────────────────────────────────
    const upload = await MediaUpload.findOne({ imageId });
    if (!upload) throw new Error(`Upload record not found: ${imageId}`);
    if (upload.status === 'completed') {
      return { skipped: true };
    }

    // ──────────────────────────────────────────────
    // 2. Update status to 'processing'
    // ──────────────────────────────────────────────
    await MediaUpload.updateOne(
      { imageId },
      { $set: { status: 'processing', updatedAt: new Date() } }
    );

    // ──────────────────────────────────────────────
    // 3. Download original image from S3
    // ──────────────────────────────────────────────
    let originalBuffer;
    try {
      const obj = await s3Client.send(new GetObjectCommand({
        Bucket: BUCKET_NAME,
        Key: s3Key,
      }));
      originalBuffer = Buffer.from(await obj.Body.transformToByteArray());
    } catch (err) {
      if (err.name === 'NoSuchKey' || err.$metadata?.httpStatusCode === 404) {
        await MediaUpload.updateOne({ imageId }, { $set: { status: 'failed', updatedAt: new Date() } });
        return { failed: true, reason: 'file_not_in_s3' };
      }
      throw err; // Transient error → BullMQ retries
    }

    // ──────────────────────────────────────────────
    // 4. Process with sharp
    // ──────────────────────────────────────────────
    const metadata = await sharp(originalBuffer).metadata();

    // Thumbnail: 300x300 cover crop, webp
    const thumbnailBuffer = await sharp(originalBuffer)
      .resize(300, 300, { fit: 'cover', position: 'centre' })
      .webp({ quality: 80 })
      .toBuffer();

    // Optimized: max 1200px longest side, preserve aspect ratio, strip EXIF, webp
    const optimizedBuffer = await sharp(originalBuffer)
      .resize(1200, 1200, { fit: 'inside', withoutEnlargement: true })
      .rotate()  // auto-rotate based on EXIF before stripping it
      .webp({ quality: 82 })
      .toBuffer();

    // ──────────────────────────────────────────────
    // 5. Upload processed versions to S3
    // ──────────────────────────────────────────────
    const thumbnailKey = `thumbs/${chatId}/${imageId}.webp`;
    const optimizedKey = `optimized/${chatId}/${imageId}.webp`;

    await Promise.all([
      s3Client.send(new PutObjectCommand({
        Bucket: BUCKET_NAME,
        Key: thumbnailKey,
        Body: thumbnailBuffer,
        ContentType: 'image/webp',
        CacheControl: 'public, max-age=31536000, immutable',
      })),
      s3Client.send(new PutObjectCommand({
        Bucket: BUCKET_NAME,
        Key: optimizedKey,
        Body: optimizedBuffer,
        ContentType: 'image/webp',
        CacheControl: 'public, max-age=31536000, immutable',
      })),
    ]);

    // ──────────────────────────────────────────────
    // 6. Build URLs and mark completed
    // ──────────────────────────────────────────────
    const baseUrl = process.env.API_BASE_URL || '';
    const thumbnailUrl = `${baseUrl}/v1/media/${imageId}?variant=thumbnail`;
    const optimizedUrl = `${baseUrl}/v1/media/${imageId}?variant=optimized`;

    await MediaUpload.updateOne(
      { imageId },
      {
        $set: {
          status: 'completed',
          thumbnailS3Key: thumbnailKey,
          optimizedS3Key: optimizedKey,
          thumbnailUrl,
          optimizedUrl,
          width: metadata.width,
          height: metadata.height,
          updatedAt: new Date(),
        },
      }
    );

    // ──────────────────────────────────────────────
    // 7. Check if ALL images for this message are complete
    // ──────────────────────────────────────────────
    const pendingCount = await MediaUpload.countDocuments({
      messageId,
      status: { $ne: 'completed' },
    });
    const allComplete = pendingCount === 0;

    // ──────────────────────────────────────────────
    // 8. Notify uploader via SSE
    // ──────────────────────────────────────────────
    emitSSEEvent(userId, {
      type: 'upload:status',
      payload: {
        imageId, messageId, chatId, status: 'completed',
        thumbnailUrl, optimizedUrl,
        width: metadata.width, height: metadata.height,
        allImagesComplete: allComplete,
      },
    });

    // ──────────────────────────────────────────────
    // 9. If all done, notify other chat participants
    // ──────────────────────────────────────────────
    if (allComplete) {
      const group = await Group.findById(chatId);
      if (group) {
        const otherMembers = group.members.filter(
          (m) => m.toString() !== userId.toString()
        );
        for (const memberId of otherMembers) {
          emitSSEEvent(memberId.toString(), {
            type: 'message:media-ready',
            payload: { messageId, chatId },
          });
        }
      }
    }

    return { completed: true, imageId };
  },
  {
    connection: REDIS_CONNECTION,
    concurrency: 5,
    limiter: { max: 30, duration: 10000 },
  }
);

// On final failure, notify the user
mediaWorker.on('failed', (job, err) => {
  console.error(`[media-worker] Job ${job.id} failed (attempt ${job.attemptsMade}/${job.opts.attempts}):`, err.message);

  if (job.attemptsMade >= job.opts.attempts) {
    const { imageId, messageId, chatId, userId } = job.data;
    MediaUpload.updateOne({ imageId }, { $set: { status: 'failed', updatedAt: new Date() } })
      .then(() => {
        emitSSEEvent(userId, {
          type: 'upload:status',
          payload: { imageId, messageId, chatId, status: 'failed' },
        });
      })
      .catch(console.error);
  }
});

export default mediaWorker;
```

### 4. SSE Integration for Upload Status

#### 4.1 SSE Connection Manager

**Single server (EventEmitter):**

```javascript
// sse/emitter.js
import { EventEmitter } from 'events';

const sseEmitter = new EventEmitter();
sseEmitter.setMaxListeners(0);

export function emitSSEEvent(userId, event) {
  sseEmitter.emit(`sse:${userId}`, event);
}

export function subscribeSSE(userId, res) {
  const handler = (event) => {
    res.write(`event: ${event.type}\n`);
    res.write(`data: ${JSON.stringify(event.payload)}\n\n`);
  };
  sseEmitter.on(`sse:${userId}`, handler);
  return () => sseEmitter.off(`sse:${userId}`, handler);
}
```

**IMPORTANT: If the worker runs in a separate process** (recommended), use **Redis Pub/Sub** instead — the EventEmitter only works within a single process:

```javascript
// sse/emitter.js (Redis Pub/Sub version)
import Redis from 'ioredis';

const pub = new Redis(process.env.REDIS_URL);
const sub = new Redis(process.env.REDIS_URL);
const localHandlers = new Map();

export function emitSSEEvent(userId, event) {
  pub.publish(`sse:${userId}`, JSON.stringify(event));
}

export function subscribeSSE(userId, res) {
  const channel = `sse:${userId}`;
  const handler = (ch, message) => {
    if (ch !== channel) return;
    const event = JSON.parse(message);
    res.write(`event: ${event.type}\n`);
    res.write(`data: ${JSON.stringify(event.payload)}\n\n`);
  };

  sub.subscribe(channel);
  sub.on('message', handler);

  if (!localHandlers.has(userId)) localHandlers.set(userId, new Set());
  localHandlers.get(userId).add(handler);

  return () => {
    sub.off('message', handler);
    const handlers = localHandlers.get(userId);
    if (handlers) {
      handlers.delete(handler);
      if (handlers.size === 0) {
        sub.unsubscribe(channel);
        localHandlers.delete(userId);
      }
    }
  };
}
```

**Use the Redis version if:** the worker is a separate process, or you have/plan multiple API instances. You already have Redis for BullMQ, so no new infrastructure is needed.

#### 4.2 SSE Endpoint

```javascript
app.get('/v1/events', authenticate, (req, res) => {
  res.writeHead(200, {
    'Content-Type': 'text/event-stream',
    'Cache-Control': 'no-cache',
    'Connection': 'keep-alive',
    'X-Accel-Buffering': 'no',    // disable nginx buffering

  const heartbeat = setInterval(() => res.write(`: heartbeat\n\n`), 15000);

  req.on('close', () => {
    clearInterval(heartbeat);
    cleanup();
  });
});
```

### 5. Reconciliation Cron Job

Catches uploads where the client PUT to S3 but never called `/complete`.

```javascript
// cron/reconcile-uploads.js
import cron from 'node-cron';

cron.schedule('*/30 * * * *', async () => {
  console.log('[reconcile] Checking for orphaned uploads...');

  const cutoff = new Date(Date.now() - 60 * 60 * 1000);

  const staleUploads = await MediaUpload.find({
    status: 'pending',
    presignedUrlExpiresAt: { $lt: new Date() },
    createdAt: { $lt: cutoff },
  }).limit(100);

  let reconciled = 0, markedFailed = 0;

  for (const upload of staleUploads) {
    try {
      await s3Client.send(new HeadObjectCommand({ Bucket: BUCKET_NAME, Key: upload.s3Key }));

      // File exists — enqueue for processing
      await mediaQueue.add('process-image', {
        imageId: upload.imageId, s3Key: upload.s3Key,
        chatId: upload.chatId.toString(), userId: upload.userId.toString(),
        messageId: upload.messageId, mimeType: upload.mimeType,
      }, { jobId: upload.imageId, attempts: 3, backoff: { type: 'exponential', delay: 2000 } });

      await MediaUpload.updateOne({ imageId: upload.imageId }, { $set: { status: 'uploaded', updatedAt: new Date() } });
      reconciled++;
    } catch (err) {
      if (err.name === 'NoSuchKey' || err.$metadata?.httpStatusCode === 404) {
        await MediaUpload.updateOne({ imageId: upload.imageId }, { $set: { status: 'failed', updatedAt: new Date() } });
        markedFailed++;
      } else {
        console.error(`[reconcile] Error checking ${upload.imageId}:`, err.message);
      }
    }
  }

  console.log(`[reconcile] Reconciled: ${reconciled}, Failed: ${markedFailed}`);
});
```

**Optional: cleanup cron for raw uploads (saves storage):**

```javascript
cron.schedule('0 3 * * *', async () => {  // daily at 3 AM
  const sevenDaysAgo = new Date(Date.now() - 7 * 24 * 60 * 60 * 1000);

  const oldCompleted = await MediaUpload.find({
    status: 'completed',
    updatedAt: { $lt: sevenDaysAgo },
    s3Key: { $regex: /^uploads\// },
  }).limit(500);

  for (const upload of oldCompleted) {
    try {
      await s3Client.send(new DeleteObjectCommand({ Bucket: BUCKET_NAME, Key: upload.s3Key }));
    } catch (err) {
      console.warn(`[cleanup] Failed to delete ${upload.s3Key}:`, err.message);
    }
  }

  console.log(`[cleanup] Deleted ${oldCompleted.length} raw uploads older than 7 days`);
});
```

### 6. S3 Bucket Configuration

**CORS (required for direct client uploads):**
```json
[
  {
    "AllowedHeaders": ["*"],
    "AllowedMethods": ["PUT"],
    "AllowedOrigins": ["*"],
    "ExposeHeaders": ["ETag"],
    "MaxAgeSeconds": 3600
  }
]
```

**Other settings:**
- Block all public access: ON
- Transfer Acceleration: enable if users are geographically spread
- Lifecycle: transition `uploads/` to Infrequent Access after 7 days, delete after 30

**S3 folder structure:**
```
your-bucket/
├── uploads/        ← raw images (temporary, cleaned after 7-30 days)

├── thumbs/         ← 300x300 webp thumbnails (permanent)
│   └── {chatId}/{imageId}.webp
└── optimized/      ← max-1200px webp (permanent)
    └── {chatId}/{imageId}.webp
```

### 11. Duplication Prevention Summary

| Layer | Mechanism |
|---|---|
| **Client** | Generates `imageId` (UUID) before upload. Same selection → same `imageId` |
| **API `/init`** | Upserts on `imageId`. Returns `alreadyComplete` if done |
| **API `/complete`** | Returns existing result if already completed/processing |
| **BullMQ** | `jobId: imageId` — same imageId won't be queued twice |
| **Worker** | Checks `status === 'completed'` before processing |
| **S3** | Same key derived from `imageId` — re-uploads overwrite |
